**Throttle Ramp:**
- `ThrottleRampTime` (float, default 1.5f) - Time to ramp throttle from 0 to 1
- `InitialThrottleLimit` (float, default 0.7f, ClampMin=0, ClampMax=1) - Maximum initial throttle

**Traction Control:**
- `bEnableTractionControl` (bool, default true)
- `WheelSlipThreshold` (float, default 0.2f, ClampMin=0, ClampMax=1) - Slip threshold before reducing torque
- `TorqueReductionFactor` (float, default 0.3f, ClampMin=0, ClampMax=1) - Amount to reduce torque when slip detected

**RPM Control:**
- `TargetLaunchRPM` (float, default 4000.0f) - Target launch RPM for clutch-style launch
- `bHoldRPMAtLaunch` (bool, default false) - Whether to hold RPM at target before launch

Override ValidateConfig() to validate reasonable ranges. Follow the exact pattern from GSDWheelConfig (include GSDVehicleConfig.h as reference for property categories).

DO NOT add any methods beyond constructor, ValidateConfig(), and GetDisplayName() - these are pure data containers.

Create UGSDTuningPreset extending UGSDDataAsset with the following properties organized by category:

**Steering:**
- `SteeringRatio` (float, default 1.0f) - Steering sensitivity multiplier
- `MaxSteeringAngle` (float, default 70.0f) - Maximum steering angle in degrees

**Suspension:**
- `SuspensionStiffness` (float, default 1.0f) - Suspension stiffness multiplier
- `SuspensionDamping` (float, default 1.0f) - Suspension damping multiplier

**Tire:**
- `FrictionMultiplier` (float, default 1.0f) - Tire friction multiplier
- `LateralStiffness` (float, default 1.0f) - Lateral stiffness multiplier

**Engine:**
- `TorqueMultiplier` (float, default 1.0f) - Engine torque multiplier
- `MaxRPM` (float, default 7000.0f) - Maximum engine RPM

**Physics:**
- `MassMultiplier` (float, default 1.0f) - Mass multiplier (1.0 = normal, >1 = heavier)
- `DragCoefficient` (float, default 0.3f) - Aerodynamic drag coefficient

Add two static helper functions:
- `GetDefaultCompliancePreset()` - Returns placeholder for future default compliance preset (return nullptr for now)
- `GetDefaultChaosPreset()` - Returns placeholder for future default chaos preset (return nullptr for now)

These will be implemented later when default presets are created in editor.

DO NOT add InterpolateToPreset - that belongs on the vehicle pawn, not the config.

**Socket Information:**
- `SocketName` (FName) - Socket name on skeletal mesh for attachment point
- `AttachmentType` (EGSDAttachmentType) - Type of attachment
- `DisplayName` (FString) - Display name for UI

**Mesh:**
- `AttachmentMesh` (TSoftObjectPtr<UStaticMesh>) - Static mesh to attach (soft reference for async loading)

**Physics:**
- `bHasCollision` (bool, default true) - Whether attachment has collision
- `AdditionalMass` (float, default 0.0f) - Additional mass in kg

**Gameplay:**
- `bAffectsPhysics` (bool, default false) - Whether attachment affects vehicle physics behavior
- `DragModifier` (float, default 0.0f) - Drag coefficient modifier

Override ValidateConfig() to check SocketName is not NAME_None and AttachmentMesh is valid.

DO NOT include any spawn/attach methods - those belong in a component.
